## **📅 4월 22일: 커뮤니티 CRUD 및 UI/UX 최적화**

**주요 과업:** 게시판 기능 안정화 및 페이지네이션 도입

- **트러블슈팅 (게시글 렌더링):**
    - **문제:** 게시글 작성 후 빈 화면(White Screen) 노출 및 데이터 정렬 오류 발생.
    - **해결:** 게시글 인덱싱(1, 2, 3...) 로직을 검토하고, 삭제 기능 인터페이스를 추가하여 CRUD 사이클 완성.
- **기능 고도화:**
    - **페이지네이션(Pagination):** 게시글 양이 많아질 경우를 대비한 페이지 분할 로직 적용 완료.
- **잔여 과제:** 새로고침 시 데이터 초기화 현상 발견 → **데이터 휘발성(Persistence) 문제** 해결을 위한 상태 관리 또는 DB 연동 검토 중.

## **📅 4월 24일: AI(Ollama) 모델 연동 및 트러블슈팅**

**주요 과업:** 식단 추천 AI 엔진 최적화 및 인코딩 오류 해결

- **AI 모델 고도화:**
    - **인코딩 해결:** 답변 출력 시 한글 깨짐 현상(Encoding Issue) 수정 시도 및 복구.
    - **UI/UX 개선:** 시스템 프롬프트가 사용자 화면에 노출되지 않도록 처리하여 깔끔한 인터페이스 구축.
- **핵심 오류 해결 (Critical Fixes):**
    1. **통신 오류(404):** Ollama AI 모델명과 프로젝트 내 설정된 호스트 주소의 불일치 확인 및 수정.
    2. **구문 오류(Syntax Error):** 모델명 호출 시 **불필요한 큰따옴표("")** 사용으로 인한 인식 오류 해결.
    3. **데이터 바인딩:** 재료 입력 시 식단 결과가 미출력되는 로직 수정 후 정상 작동 확인.

## **🛠 핵심 기술 스택 요약 (Tech Stack)**

**1. React (Frontend Library)**

- **역할:** 컴포넌트 기반 UI 개발.
- **활용:** 데이터 변화에 따른 실시간 화면 렌더링 및 상태(State) 관리.

**2. ProtectedRoute (Auth Guard)**

- **역할:** 비로그인 사용자의 접근을 차단하는 **보안 관문**.
- **활용:** `/routine`, `/calendar` 등 사용자 전용 페이지 접속 시 로그인 여부를 체크하여 자동 리다이렉트(Redirect).

**3. ServiceLayout (Common UI)**

- **역할:** 서비스 전체에 적용되는 **공통 레이아웃**.
- **활용:** 다크 모드(검정 배경), 포인트 컬러(파스텔 보라)가 적용된 GNB(상단 메뉴)와 챗봇 위치를 고정하여 일관된 UX 제공.

**4. Ollama AI (Local LLM)**

- **역할:** 온디바이스 기반의 **식단 추천 엔진**.
- **활용:** 사용자가 입력한 재료를 바탕으로 맞춤형 식단을 생성하고, 404 오류 및 모델명 인식 오류(큰따옴표 이슈)를 해결하여 안정화 완료.

**5. Pagination (Data Handling)**

- **역할:** 커뮤니티 게시글 **데이터 분할 처리**.
- **활용:** 많은 양의 게시글을 사용자 화면에 과부하 없이 노출하고, 1-2-3 순차적 페이징을 통해 탐색 편의성 제공.
